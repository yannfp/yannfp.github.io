The curriculum covers the core pillars of Java and the JVM (Java Virtual Machine) environment.

### 1. The Four Pillars of OOP
* **Encapsulation:** Learning to hide the internal state of objects using access modifiers (`private`, `protected`, `public`) and getters/setters.
* **Inheritance:** Designing hierarchical class structures to reuse code and represent real-world relationships.
* **Polymorphism:** Mastering method overriding and overloading to write flexible, extensible code.
* **Abstraction:** Utilizing `abstract` classes and `interfaces` to define strict contracts for software modules.



### 2. The Java Ecosystem
* **Generics:** Implementing type-safe data structures that work across different object types.
* **Exception Handling:** Using `try-catch-finally` blocks to build resilient software that handles runtime errors gracefully.
* **Collections Framework:** Mastering `List`, `Set`, `Map`, and the `Stream API` for efficient data manipulation.
* **Unit Testing:** Writing **JUnit** tests to validate code logic‚Äîa shift from the manual test suites used in the C Piscine.

### 3. Design Patterns & Best Practices
The Piscine introduced me to common architectural patterns:
* **Singleton:** Ensuring a class only has one instance.
* **Factory:** Decoupling object creation from the main logic.
* **Strategy:** Enabling an object's behavior to be selected at runtime.
